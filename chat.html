Can you explain how connection pooling works in Postgres?
A

Connection pooling reuses a fixed set of open database connections instead of opening a new one per request. This matters because establishing a Postgres connection is expensive — it forks a backend process and does auth/TLS handshake.

Typical setup:

PgBouncer (transaction mode)
  → App holds logical connections
  → PgBouncer multiplexes onto real Postgres connections

Use transaction pooling mode for most web apps, session mode only if you rely on session-level features like advisory locks or prepared statements.
